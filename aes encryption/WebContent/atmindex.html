AES ENCRYPTION AND DECRYPTION TOOL

1.It is based on aes encryption decryption algorithm also called Rijndaels algorithm.

2.It has blocks of size 128 bits each,but with three different key lengths of 128,192 and 256.

3.The key size used for an AES cipher specifies the number of repetitions of transformation rounds that convert the input, called the plaintext, into the final output, called the ciphertext. The number of cycles of repetition are as follows:
10 cycles of repetition for 128-bit keys.
12 cycles of repetition for 192-bit keys.
14 cycles of repetition for 256-bit keys.

4.It has rounds such as:
a.key expansions
b.sub bytes
c.shift rows
d.mixed columns

5.There are different aes modes of operation like
CBC,ECB,CTR,OCB,CFB

6.The mode currently used is CTR.

7.Counter mode operation:
1) Load the initial counter value in the top register which is the same for both the sender and the receiver. It plays the same role as the IV in CFB (and CBC) mode.
2) Encrypt the contents of the counter with the key and place the result in the bottom register.
3)Take the first plaintext block P1 and XOR this to the contents of the bottom register. The result of this is C1. Send C1 to the receiver and update the counter.
4)Continue in this manner until the last plaintext block has been encrypted.
5)The decryption is the reverse process. The ciphertext block is XORed with the output of encrypted contents of counter value. After decryption of each ciphertext block counter is updated as in case of encryption.
Implementation of Counter Mode:
1)Find key using the password and in turn use the key to generate key schedule
2)First 8 bytes are filled with nonce(random number generated using date.gettime())
3)Fill the last 8 byte with a counter value
4)This block is ciphered with the key schedule
5)The result is XORed with the block of plain text and the nonce value is appended at the beginning
6)The decryption is the reverse process.

8.Encryption and decryption is done on different kinds of data such as number,text,video,files,audio and images.

11.Advantages of using AES:

1) AES is more secure (it is less susceptible to cryptanalysis than 3DES).
2) AES supports larger key sizes than 3DES's 112 or 168 bits.
3) AES is faster in both hardware and software.
4) AES is required by the latest U.S. and international standards.
5) It does not have message dependency and hence a ciphertext block does not depend on the previous plaintext blocks.
6) It has nonce, counter to produce different ciphers for the same key
7) It does not propagate error of transmission at all.


12.Disadvantages of using AES:

It requires a synchronous counter at sender and receiver. Loss of synchronization leads to incorrect recovery of plaintext.
